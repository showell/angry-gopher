## Porting recipe

For each op:

1. Open the Go emitter, identify which `Scenario` fields
   it reads (look for `sc.Helper`, `sc.Expect.Loc`,
   `sc.Board`, etc.).
2. Look at the corresponding `.dsl` file(s) to confirm
   what's actually in those scenarios.
3. Write a `verifyX : Dsl.Scenario -> Expectation`
   function in `ConformanceTests.elm`. Read from
   `sc.helper`, `sc.expect`, `sc.otherScalars`, etc.
4. Add a `case "op_name" -> verifyX sc` line to
   `verify`.
5. Run `ops/check-conformance --skip-slow` — both the
   new tests and the legacy tests should pass.
6. Spot-check: a few legacy stub-tests for this op now
   have real coverage in `ConformanceTests.elm`. The
   legacy stub IS still running (just doing the same
   assertion or nothing) — that's fine for transition.

## Gotchas

- **Card tokens carry state markers**: `8H*` =
  FreshlyPlayed, `8H**` = FreshlyPlayedByLastPlayer.
  `parseCardToken` currently strips these; if a verifier
  needs state (referee tests likely do), it'll need to
  re-parse the raw `otherScalars`-stored value OR we
  enrich the parser to preserve state. Pick the simpler
  path per verifier.
- **`otherScalars` / `otherBlocks` are typed `Dict String
  String`** — verifiers do their own `String.toInt` etc.
  No premature typing.
- **`expect` is `Expect`**: either `ExpectScalar "ok"`
  (no children), `ExpectBlock dict` (block with named
  sub-fields), or `ExpectEmpty`. Each verifier knows
  which form its op produces.
- **Don't forget the embedder**: after editing a `.dsl`,
  re-run `ops/embed_dsls_for_elm.ts` (or just
  `ops/check-conformance` which does it for you).
